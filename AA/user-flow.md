# 사용자 흐름도 (AA)

## 1. 메인 사용자 시나리오
1. 사용자가 회의 파일 업로드
2. 시스템이 회의 레코드 생성 + S3 저장
3. 백엔드가 AI 처리 파이프라인 시작
4. 사용자는 목록/상세 화면에서 처리 상태 확인
5. 처리 완료 후 요약/개인별 To-Do 확인
6. Archive에서 검색/필터/정렬로 과거 기록 조회

---

## 2. 상태 전이 기준 사용자 흐름
- `UPLOADED` → 업로드 완료
- `TRANSCRIBING` → 음성 텍스트 변환 중
- `SUMMARIZING` → 요약 생성 중
- `TODO_EXTRACTING` → 개인별 To-Do 추출 중
- `COMPLETED` → 상세 결과 조회 가능
- `FAILED` → 오류 안내 + 재처리

---

## 3. 화면 이동 흐름
- 업로드 화면 → (업로드 성공) → 회의 상세 화면
- 상세 화면 ↔ 목록(Archive) 화면
- 목록 화면 → 특정 회의 클릭 → 상세 화면

---

## 4. 예외 시나리오
- 업로드 실패: 실패 사유 표시 + 재업로드 유도
- AI 처리 실패: 상태 `FAILED` 노출 + 재처리 버튼
- 데이터 없음: 빈 상태(Empty State) 안내
